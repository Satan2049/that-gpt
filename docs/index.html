Self-hosted chat.
Your keys never leave the server.

A small React UI and Fastify API for any OpenAI-compatible provider. Conversations and prompt presets live as JSON on disk—run it locally, point it at OpenAI, Azure, or Ollama, and start chatting.

Get started View source

Built for local, private use

File-backed storage

One JSON file per conversation and preset under server/data/. No Postgres, no Redis—easy to back up and inspect.

Vision support

Attach JPEG, PNG, or WebP images to messages. The server validates type, size, and count before forwarding to your provider.

Prompt presets

Save system prompts with per-preset model and temperature. Switch presets per conversation from the sidebar.

Server-side API key

The browser talks only to your Fastify backend. Provider credentials stay in server/.env, not in the client bundle.

Light & dark theme

Toggle appearance from the header. Your choice persists in localStorage in the browser.

Clear errors

Timeouts, rate limits, invalid keys, and network failures surface as readable messages in the UI.

Architecture

client/   React + Vite + Zustand   →   proxies /api to server
server/   Fastify + TypeScript    →   file storage + AI proxy

Browser ──HTTP──► Fastify ──HTTPS──► OpenAI-compatible API
└── reads/writes JSON under server/data/

Quick start

  1. Prerequisites — Node.js 20+ and an API key for your provider. 
    git clone https://github.com/Satan2049/chatter-box.git
cd chatter-box
  2. Install 
    npm install --prefix server
npm install --prefix client
  3. Configure 
    cp server/.env.example server/.env
# Set AI_API_KEY (and optionally AI_BASE_URL, AI_MODEL)
  4. Run — API on port 3001, UI on 5173. 
    npm run dev:server   # terminal 1
npm run dev:client   # terminal 2

Open http://127.0.0.1:5173 and send your first message. Full docs: README.
