Product overview

Local chat interface with server-side API credentials

Chatterbox pairs a React client with a Fastify API proxy. Conversation history and prompt presets persist as JSON on disk. Configure any compatible provider—OpenAI, Azure OpenAI, or Ollama—and run entirely on your infrastructure.

Quick start Documentation Source code
Credentials isolated AI_API_KEY is read only by the Fastify server, never exposed to the browser bundle.
No database layer Threads and presets are plain JSON under server/data/ for simple backup and inspection.
Provider-agnostic Point AI_BASE_URL at any OpenAI-compatible endpoint with configurable model defaults.

Capabilities

Everything required for focused local use

Conversation management

Create, rename, and delete threads. History is loaded on demand from per-file JSON records.

Prompt presets

Define system prompts with model and temperature. Assign a preset per conversation from the sidebar.

Multimodal input

Attach JPEG, PNG, or WebP images. The API validates MIME type, size, and count before forwarding.

Theme persistence

Light and dark modes mirror the in-app palette. Preference is stored in browser local storage.

Operational errors

Timeouts, rate limits, authentication failures, and network issues return actionable UI messages.

Minimal dependencies

Node.js 20+, React with Vite, and Fastify. No Redis, Postgres, or external session store.

System design

Two-tier layout

Browser (React) → Fastify API → Provider API

Vite dev server proxies /api to the backend. Production builds serve static assets separately; the API remains the single integration point for AI calls and file I/O.

client/
React · Vite · Zustand → UI state, chat & preset forms
server/
Fastify · TypeScript → routes, validation, AI client, repositories

Installation

Quick start

  1. Step 1 Clone and install 
    git clone https://github.com/Satan2049/chatter-box.git
cd chatter-box
npm install --prefix server
npm install --prefix client
  2. Step 2 Configure environment 
    cp server/.env.example server/.env
# Required: AI_API_KEY
# Optional: AI_BASE_URL, AI_MODEL, AI_DEFAULT_SYSTEM_PROMPT
  3. Step 3 Run development servers 
    npm run dev:server   # http://127.0.0.1:3001
npm run dev:client   # http://127.0.0.1:5173
Security: Chatterbox does not implement authentication. Deploy on localhost or place the service behind your own network controls and reverse proxy policies.
