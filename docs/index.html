Product overview

Desktop chat with credentials kept in the Rust shell

ChatNest pairs a React UI with a Rust + Tauri backend. Conversation history and prompt presets persist as JSON on disk. Streaming replies, in-app settings, and export to Markdown or JSON — configure any compatible provider and run locally.

Download release GitHub
Credentials isolated AI_API_KEY is read only by the Rust backend, never exposed to the UI bundle.
No database layer Threads and presets are plain JSON in the app data directory for simple backup and inspection.
Provider-agnostic Point AI_BASE_URL at any OpenAI-compatible endpoint with configurable model defaults.

Capabilities

Everything required for focused local use

Conversation management

Create, rename, and delete threads. History is loaded on demand from per-file JSON records.

Prompt presets

Define system prompts with model and temperature. Assign a preset per conversation from the sidebar.

Multimodal input

Attach JPEG, PNG, or WebP images. The backend validates MIME type, size, and count before forwarding.

Streaming responses

Assistant messages render token-by-token as the provider streams the completion.

In-app settings

API key, base URL, model, system prompt, timeout, and retries — editable from Settings.

Export conversations

Download the active thread as Markdown or JSON from the header export menu.

Theme persistence

Light and dark modes mirror the in-app palette. Preference is stored in browser local storage.

Operational errors

Timeouts, rate limits, authentication failures, and network issues return actionable UI messages.

Native desktop

React with Vite in a Tauri window. Rust handles storage, validation, and provider calls.

System design

Desktop shell layout

WebView (React) → Rust (Tauri) → Provider API

The UI calls Rust via Tauri invoke. The backend is the single integration point for AI calls and file I/O.

client/
React · Vite · Zustand → UI state, chat & preset forms
src-tauri/
Rust · Tauri → commands, validation, AI client, repositories

Installation

Quick start

  1. Option A Download a release 
    # GitHub Releases → portable .exe or setup installer
# Verify SHA256.txt from the same release
  2. Option B Build from source 
    git clone https://github.com/Satan2049/chat-nest.git
cd chat-nest
npm install
npm run build
  3. Configure API access 
    # Open Settings in the app, or edit:
# %APPDATA%\com.chatnest.desktop\.env
#
# AI_API_KEY, AI_BASE_URL, AI_MODEL, …
  4. Develop Run with hot reload 
    npm run dev
Security: ChatNest is a single-user local desktop app. Report vulnerabilities via SECURITY.md.
